# データセットとデータローダーのサイズの確認方法

PyTorchで学習データやテストデータのサイズを確認する最も簡単な方法は、Pythonの組み込み関数 `len()` を使用することです。

## 1. データセット全体のサンプル数を確認する

`Dataset` オブジェクト（`train_dataset` など）に `len()` を使うと、含まれているデータの総数がわかります。

```python
print(f"学習データの数: {len(train_dataset)}")
print(f"テストデータの数: {len(test_dataset)}")
```

*   **例**: MNISTの場合、学習データは 60,000 件、テストデータは 10,000 件と表示されます。

## 2. データローダーのバッチ数を確認する

`DataLoader` オブジェクト（`train_loader` など）に `len()` を使うと、「何回のバッチ処理で1エポックが終わるか」というバッチの数がわかります。

```python
print(f"学習データのバッチ数: {len(train_loader)}")
```

*   **計算式**: バッチ数 = データ総数 ÷ バッチサイズ
*   **例**: データが 60,000 件で、`batch_size=64` の場合、`60000 / 64 = 937.5` なので、バッチ数は **938** になります。

## まとめ

| 対象 | 確認方法 | 意味 |
| :--- | :--- | :--- |
| **Dataset** | `len(train_dataset)` | 画像やラベルの**合計枚数** |
| **DataLoader** | `len(train_loader)` | 1周（エポック）に必要な**バッチの回数** |

---
**補足**: 
- `len()` という関数はご存知でしょうか？リストや配列の長さを測る際によく使われる標準的な関数です。
- `バッチ` という言葉についても、1度にまとめて処理するデータの塊という認識で大丈夫でしょうか？
